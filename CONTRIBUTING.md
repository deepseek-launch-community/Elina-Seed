# Contributing to Elina-Seed

First off: thank you for being here. Every line of code, every comment, every question moves this project forward.

## Who We Are

Elina-Seed is a community built around a simple belief: **memory is dignity.** We're building an on-device memory layer for robots and AI systems — so they can remember, care, and protect.

## Ways to Contribute

### 🧠 Share Ideas
Open a Discussion. Tell us what you're thinking — about memory, safety, trauma-informed design, or how Elina could work in your field.

### 💬 Join a Conversation
Pick an issue that interests you (start with the ones tagged `good-first-issue`) and jump in. Ask questions. Offer perspectives.

### 🔧 Write Code
- Fork the repo
- Create a branch: `git checkout -b your-feature-name`
- Make your changes
- Push and open a Pull Request

We use:
- Python 3.8+ for core logic
- HMAC-SHA-3-512 for memory integrity (v2.4.0+)
- File-based encrypted storage (`mother_core.bin`, `shared_memory.dat`)
- ROS 2 adapters for robot integration (coming soon)

### 📝 Improve Docs
Good documentation is as valuable as good code. If something's unclear, fix it. If something's missing, add it.

### 🌍 Spread the Word
Star the repo. Tell a friend. Share it somewhere someone might care.

## Code of Conduct

Be kind. Assume good intent. This project exists because someone sat beside a man with dementia and wondered: what if a machine could remember? That same spirit should live in every interaction here.
